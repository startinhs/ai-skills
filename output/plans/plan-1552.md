# Plan Fix — Issue 1552
**ProductsListView: hardcode MaxResultCount=1000 gây lỗi validation "không được nhiều hơn 10000"**

## Thông tin
- Branch: `fix/fix-product-1552-tinhlm`
- Base: `release/1.0.0-avntt-rc1`
- Priority: Medium | Type: Issue

## Root Cause

Trong `ProductsListView.razor.cs`, hai chỗ build filter để gọi `GetListSimpleForListViewAsync` sử dụng `MaxResultCount = 1000` hardcoded. Khi server cấu hình `MaxAllowedResultCount` khác với 1000 (ví dụ thấp hơn), hoặc có validation check trong AppService, sẽ gây lỗi `"MaxResultCount không được nhiều hơn X"`.

**File:** `src/.../Pages/MasterData/Product/ProductsListView.razor.cs`

**Trước fix:**
```csharp
// Chỗ 1 — load theo hierarchy node:
MaxResultCount = 1000 // bounded load for special nodes

// Chỗ 2 — load all products:
MaxResultCount = 1000 // Like Customer
```

**Sau fix:**
```csharp
// Chỗ 1:
MaxResultCount = Filter.MaxAllowedResultCount

// Chỗ 2:
MaxResultCount = Filter.MaxAllowedResultCount
```

## Fix

**File:** `src/HQSOFT.Xspire.Application.Blazor/Pages/MasterData/Product/ProductsListView.razor.cs`

Tìm và thay thế 2 chỗ:
```csharp
// Thay:

// Bằng:
MaxResultCount = Filter.MaxAllowedResultCount
```
