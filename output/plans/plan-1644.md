# Plan Fix — Issue 1644
**Đơn hàng bị chuyển Pending sai khi CreditLimit = 0 (khách hàng chưa cấu hình hạn mức SAP)**

## Thông tin
- Branch: `fix/fix-SO-1644-tinhlm`
- Base: `release/1.0.0-avntt-rc1`
- Priority: High | Type: Issue

## Root Cause

Hàm kiểm tra hạn mức tín dụng trong `Action.razor.cs` và `SalesOrder.razor.cs` không phân biệt trường hợp khách hàng **chưa được cấu hình CreditLimit trong SAP** (CreditLimit = 0) với trường hợp **đã vượt hạn mức**. Kết quả: khách hàng không có hạn mức vẫn bị block/pending khi `remainingAmount < 0`.

**Files:**
- `src/.../Pages/OrderManagement/SalesOrder/Action.razor.cs`
- `src/.../Pages/OrderManagement/SalesOrder/SalesOrder.razor.cs`

**Logic SAI (trước fix):**
```csharp
// Vượt hạn thanh toán và vượt hạn mức nợ
if (sapResult.OverdueCheckResult && remainingAmount < 0)
    // block → SAI: CreditLimit=0 → remainingAmount = 0 - AmountIncludeSo < 0 → bị block sai

// Vượt hạn mức nợ
if (remainingAmount < 0)
    // block → SAI: tương tự
```

**Logic ĐÚNG (sau fix):**
```csharp
// Chỉ kiểm tra khi CreditLimit > 0 (KH có cấu hình hạn mức trong SAP)
if (sapResult.OverdueCheckResult && sapResult.CreditLimit > 0 && remainingAmount < 0)
    // block

if (sapResult.CreditLimit > 0 && remainingAmount < 0)
    // block
```

## Fix

**Bước 1 — `Action.razor.cs` (2 chỗ trong `CheckCreditLimitAsync`):**
```csharp
// SAI:
if (sapResult.OverdueCheckResult && remainingAmount < 0)
// ĐÚNG:
if (sapResult.OverdueCheckResult && sapResult.CreditLimit > 0 && remainingAmount < 0)

// SAI:
if (remainingAmount < 0)
// ĐÚNG:
if (sapResult.CreditLimit > 0 && remainingAmount < 0)
```

**Bước 2 — `SalesOrder.razor.cs` (2 chỗ tương tự):**
```csharp
// Áp dụng cùng pattern — thêm sapResult.CreditLimit > 0 trước kiểm tra remainingAmount
```
