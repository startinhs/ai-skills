# Plan Fix — Request 1413
**Field "Đơn hàng thay thế" hiển thị mã PSI thay vì mã SO**

## Thông tin
- Branch: `fixbug-issue-1413/tinhlm`
- Base: `release/1.0.0-avntt-rc1`
- Priority: Medium | Type: Request

## Yêu cầu
Khi user chọn đơn hàng thay thế (popup), chọn theo mã PSI. Sau khi xác nhận, field "Đơn hàng thay thế" phải hiển thị **mã PSI** đã chọn (ví dụ: PSI0000001158), không phải mã SO (SO0000008177).

## Root Cause (Confirmed từ code + DB)

**Từ DB:** PSINumber của regular SO = "PSI0000001158", OrderNumber = "SO0000008170". Đây là 2 trường khác nhau.

**File:** `src/HQSOFT.Xspire.Application.Blazor/Pages/OrderManagement/SalesOrder/SalesOrder.razor.cs`

**`OnOk()` — line ~14733 (SAI):**
```csharp
EditingDoc.ReplacementOrderNumber = SalesOrderSROModel.OrderNumber;  // ← hiển thị mã SO
```

**`OnReplacementOrderPopupConfirmed()` — line ~15050 (cần verify):**
```csharp
SalesOrderSROModel.OrderNumber = selectedOrder.SalesVoucherCode;   // SO code
SalesOrderSROModel.PSINumber = selectedOrder.ReturnVoucherCode;    // ← cần verify: có phải PSINumber không?
```

**⚠️ Cần verify khi implement:**
- `selectedOrder` là `SalesOrderDto` — check xem `SalesOrderDto.PSINumber` có tồn tại không
- Nếu có: đổi mapping sang `selectedOrder.PSINumber`
- Nếu không: check `ReturnVoucherCode` vs `PSINumber` trong DB context của replacement order

## Fix

**Bước 1 — `OnOk()` (bắt buộc):**
```csharp
// TRƯỚC:
EditingDoc.ReplacementOrderNumber = SalesOrderSROModel.OrderNumber;

// SAU:
EditingDoc.ReplacementOrderNumber = SalesOrderSROModel.PSINumber;
```

**Bước 2 — `OnReplacementOrderPopupConfirmed()` (verify + fix nếu cần):**
```csharp
// Nếu SalesOrderDto có PSINumber field:
SalesOrderSROModel.OrderNumber = selectedOrder.OrderNumber;    // SO code (giữ để lưu Id)
SalesOrderSROModel.PSINumber = selectedOrder.PSINumber;        // PSI code để hiển thị

// Nếu không có PSINumber field trong DTO, cần thêm mapping vào AppService
```
