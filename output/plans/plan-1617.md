## Current Behavior (observed from code)
Khi SRO (đơn trả hàng theo xe WF_VS) đạt trạng thái "Ghi sổ":
1. `TradeDiscountEventBus` → `RefundTradeDiscountPaymentBySalesOrderIdAsync(SRO.Id)`
2. `GetDataBySalesOrderId(SRO.Id)` → lấy `SalesOrderTradeDiscount` của **đơn SRO**, không phải đơn gốc
3. SRO có thể có trade discount records với `BonusLineId` trỏ vào **sai CT CKTM**
4. → `BonusLine.AccruedAmount` bị giảm trên **CT CKTM sai**

## Expected Behavior
Khi SRO trả hàng:
- Phải hoàn lại đúng CT CKTM đã được tích lũy lúc ghi sổ đơn gốc (WF_VS)
- Dùng `SalesReturnOrderId` (ID đơn gốc) để tìm `SalesOrderTradeDiscount` đúng
- `BonusLine.AccruedAmount` của **đúng CT CKTM** được giảm đi

## Root Cause
**File:** `modules/hqsoft.xspire.ordermanagement/src/HQSOFT.Xspire.OrderManagement.Application/BonusLines/BonusLineAppService.Extended.cs`

Method `RefundTradeDiscountPaymentBySalesOrderIdAsync` — dùng sai ID để lookup:
```csharp
// BUG: salesOrderId = SRO.Id → SalesOrderTradeDiscount của SRO có BonusLineId sai CT
var salesOrderTradeDiscount = await _salesOrderTradeDiscountRepository.GetDataBySalesOrderId(salesOrderId);
```

Với SRO, cần dùng `SalesReturnOrderId` (đơn gốc):
- `SalesReturnOrderId` được set tại `CancelSaleOrderAsync`: `newSalesOrder.SalesReturnOrderId = salesOrdersInfo.Id`
- Đơn gốc (WF_VS) có `SalesOrderTradeDiscount` với `BonusLineId` đúng → tích lũy đúng CT CKTM lúc ghi sổ

## Fix
**Vị trí:** Đầu `RefundTradeDiscountPaymentBySalesOrderIdAsync`, thay thế query:

```csharp
// BEFORE (sai):
var salesOrderTradeDiscount = await _salesOrderTradeDiscountRepository.GetDataBySalesOrderId(salesOrderId);

// AFTER (đúng): dùng ID đơn gốc nếu là SRO
var originalOrderId = (salesOrder.SalesReturnOrderId.HasValue)
    ? salesOrder.SalesReturnOrderId.Value
    : salesOrderId;
var salesOrderTradeDiscount = await _salesOrderTradeDiscountRepository.GetDataBySalesOrderId(originalOrderId);
```
