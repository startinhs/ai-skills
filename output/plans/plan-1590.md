# Plan Fix — Issue 1590
**Invoice Mapping: sản phẩm PA hàng bán bị ẩn sai trong Case 3 (Mua PA tặng PA)**

## Thông tin
- Branch: `fix/fix-invoice-1590-tinhlm`
- Base: `release/1.0.0-avntt-rc1`
- Priority: High | Type: Issue

## Root Cause

**File:** `modules/hqsoft.sap.dmsintegration/src/HQSOFT.SAP.DMSIntegration.Application/InvoiceMapping/Services/SalesOrderToInvoiceMapper.cs`

Trong vòng lặp build danh sách items hóa đơn, có logic:
```csharp
if (!product.IsFreeItem)
{
    // Case 3 (Mua PA tặng PA): PA hàng bán (IsPA=true) => ẩn dòng cha,
    // chỉ hiện sản phẩm rã từ ProductComponent (do BuildPAExpandedItemsAsync xử lý).
    if (product.IsPA && isSoPackaging && isSoGiftProduct)
    {
        continue;  // ← ẩn dòng cha hàng bán
    }
    productItemsNormal.Add(item);
}
```

**Bug:** Dòng cha PA hàng bán bị `continue` (ẩn) nhưng hóa đơn cần hiển thị dòng cha để user biết sản phẩm nào được mua. `BuildPAExpandedItemsAsync` chỉ rã component KM (IsPromotion), không thay thế dòng cha hàng bán.

## Fix

```csharp
if (!product.IsFreeItem)
{
    // Hàng bán: PA hàng bán trong Case 3 vẫn giữ dòng cha để hiện trên HĐ.
    // BuildPAExpandedItemsAsync sẽ rã riêng component KM (IsPromotion) vào phần KKKNT.
    productItemsNormal.Add(item);  // ← không skip nữa
}
```

Xóa toàn bộ block `if (product.IsPA && isSoPackaging && isSoGiftProduct) { continue; }`.

## Phạm vi thay đổi
- **1 file:** `SalesOrderToInvoiceMapper.cs`
- **~6 dòng** (if block + continue) bị xóa
- Không cần migration

## Lưu ý
- Sau fix: hóa đơn Case 3 sẽ hiện cả dòng cha (hàng bán PA) + các component KM từ `BuildPAExpandedItemsAsync`
- Cần test với đơn hàng có IsPA=true + IsSoPackaging + IsSoGiftProduct
