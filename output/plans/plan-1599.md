# Plan Fix — Issue 1599
**SalesOrderHistory: hiển thị sai cột mã sản phẩm và xử lý UOM chưa đúng**

## Thông tin
- Branch: `fix/fix-SO-1599-tinhlm`
- Base: `release/1.0.0-avntt-rc1`
- Priority: Medium | Type: Issue

## Root Cause

**File:** `src/.../Pages/OrderManagement/SalesOrderHistory/SalesOrderHistory.razor`

Grid hiển thị sản phẩm trong lịch sử đơn hàng dùng `FieldName="@nameof(SalesOrderProductUpdateDto.ProductId)"` (hiển thị Guid) cho cột "MÃ SẢN PHẨM". Phải dùng `ProductCode` thay vì `ProductId`.

Ngoài ra, CellDisplayTemplate cho cột này cần được cập nhật để hiển thị đúng `ProductCode` thay vì lookup theo Id.

## Fix

**File:** `SalesOrderHistory.razor`

Cột "MÃ SẢN PHẨM":
```razor
// SAI:
<DxGridDataColumn FieldName="@nameof(SalesOrderProductUpdateDto.ProductId)" Caption="@L["MÃ SẢN PHẨM"]">

// ĐÚNG:
<DxGridDataColumn FieldName="@nameof(SalesOrderProductUpdateDto.ProductCode)" Caption="@L["MÃ SẢN PHẨM"]">
```

CellDisplayTemplate cũng cần cập nhật để bind đúng `ProductCode`.

## Phạm vi thay đổi
- **1 file:** `SalesOrderHistory.razor`
- **Column definition** — đổi `ProductId` → `ProductCode`
- Không cần migration
