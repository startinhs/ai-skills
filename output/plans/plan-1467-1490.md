# Plan Fix — Issue 1467 / 1490
**SalesOrdersListView: TextChanged trigger fetch không cần thiết khi xóa từng ký tự**

## Thông tin
- Branch: `fix/fix-SO-1467-1490-tinhlm`
- Base: `release/1.0.0-avntt-rc1`
- Priority: Medium | Type: Issue

## Root Cause

Tất cả các TextBox filter trong `SalesOrdersListView.razor` đều có handler:
```razor
TextChanged="@(async (string value) => { Filter.X = value; if (string.IsNullOrEmpty(value)) await GetDataAsync(false); })"
```

**Bug:** Handler gọi `GetDataAsync` khi `value` empty. Điều này xảy ra **mỗi lần user xóa ký tự** khi field trống rồi xóa tiếp (hoặc clear button). Nếu field đã rỗng trước đó (prev = ""), việc xóa lại trigger thêm 1 lần fetch thừa.

**Ảnh hưởng:** 5 TextBox filters (OrderNumber, InvoiceNumber, CustomerName, IdentificationNumber, WorkUnitCode, TaxCode) đều bị lỗi này → gây thêm n lần API call vô ích khi user tương tác với filter.

## Fix

Chỉ fetch khi field **thực sự thay đổi từ có giá trị → rỗng** (prev không rỗng):

```razor
// SAI:
TextChanged="@(async (string value) => { Filter.OrderNumber = value; if (string.IsNullOrEmpty(value)) await GetDataAsync(false); })"

// ĐÚNG:
TextChanged="@(async (string value) => { var prev = Filter.OrderNumber; Filter.OrderNumber = value; if (string.IsNullOrEmpty(value) && !string.IsNullOrEmpty(prev)) await GetDataAsync(false); })"
```

**Áp dụng cho tất cả 5–6 TextBox filters** trong file `.razor`.

## Phạm vi thay đổi
- **1 file:** `SalesOrdersListView.razor`
- **5–6 TextChanged handlers** — thêm `var prev = ...; !string.IsNullOrEmpty(prev)` check
- Không cần migration
