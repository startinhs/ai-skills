# Plan Fix — Issue 1440
**SalesOrder: catch block trong CheckCreditLimit gọi SetPendingOrder sai và có logic thừa**

## Thông tin
- Branch: `fix/fix-SO-1440-tinhlm`
- Base: `release/1.0.0-avntt-rc1`
- Priority: High | Type: Issue

## Root Cause

**File:** `src/.../Pages/OrderManagement/SalesOrder/Action.razor.cs`

Trong `CheckCreditLimitAsync`, block `try` bao gồm 3 nhánh kiểm tra credit limit. Tuy nhiên, các nhánh này đã gọi `SetPendingOrderAsync()` trước khi `return false`. **Vấn đề:** trong block `catch`, code lại gọi thêm một lần SAP API mới + `SetPendingOrderAsync()` + cập nhật SalesOrder → đây là logic thừa, nguy hiểm vì khi exception xảy ra, double-update order có thể gây data inconsistency.

**Block catch SAI (trước fix):**
```csharp
catch (Exception ex)
{
    var request = new RequestCreditLimitModel(...);
    var list = new CREDIT_LIMIT(); list.DT_RP_CREDIT_LIMIT.Add(request);
    var response = await CreditLimitAppService.GetCreditLimitAsync(list);  // ← gọi SAP lần 2!
    var SalesOrderPending = await SalesOrdersAppService.GetAsync(EditingDoc.Id);
    SalesOrderPending.IsPendingOrder = true;
    // ... update các fields
    await SalesOrdersAppService.UpdateAsync(...);  // ← update khi đang trong exception
    await UiMessageService.Error($"Lỗi khi kiểm tra Credit Limit: {ex.Message}");
    return false;
}
```

**Sau fix (ĐÚNG — chỉ log error):**
```csharp
catch (Exception ex)
{
    await UiMessageService.Error($"Lỗi khi kiểm tra Credit Limit: {ex.Message}");
    return false;
}
```

Đồng thời, `SetPendingOrderAsync()` cũng bị xóa khỏi các nhánh `if` trong try (pending được xử lý riêng sau khi return).

## Phạm vi thay đổi
- **1 file:** `Action.razor.cs`
- **~30 dòng** trong catch block bị xóa
- **3 lần gọi** `SetPendingOrderAsync()` trong try bị xóa
- Không cần migration
