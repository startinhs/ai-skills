## Yêu cầu
Sort order của sản phẩm trong Báo cáo tồn kho cuối ngày trên App (SFA mobile) phải giống như sort order trên Web (daily inventory report).

## Root Cause

**App function (SAI):**
File: `src/HQSOFT.Xspire.Application.EntityFrameworkCore/Scripts/fs_rp_sfainventoryofsalesteam.psql`
```sql
ORDER BY sorted_data."ProductCode";  -- chỉ sort theo ProductCode
```

**Web function (ĐÚNG - cần theo):**
File: `src/HQSOFT.Xspire.Application.EntityFrameworkCore/Scripts/fs_rp_dailyinventoryofsalesteam.psql`
```sql
ORDER BY
    final_result."WareHouse",
    CASE WHEN final_result."HierarchyL02Code" IS NULL OR final_result."HierarchyL02Code" = '' THEN 1 ELSE 0 END,
    final_result."HierarchyL02Code",
    final_result."HierarchyL03Code",
    CASE
        WHEN final_result."Attribute03" = 'Z002' THEN 0
        WHEN final_result."Attribute03" = 'Z013' THEN 1
        WHEN final_result."Attribute03" = 'Z009' THEN 2
        ELSE 3
    END,
    final_result."type_description",
    final_result."HierarchyL05Code",
    final_result.product_code,
    final_result."Level" ASC;
```

## Fix

**Bước 1:** Đọc `fs_rp_sfainventoryofsalesteam.psql` để xác định alias/column names có khớp không
**Bước 2:** Update ORDER BY trong `fs_rp_sfainventoryofsalesteam.psql` để match sort order của `fs_rp_dailyinventoryofsalesteam.psql` (điều chỉnh alias nếu khác nhau)
**Bước 3:** Tạo EF Core migration để deploy function mới lên DB
**Bước 4:** Chạy `inject_multi_psql.py` để inject vào migration
**Bước 5:** Yêu cầu user build + run migration
