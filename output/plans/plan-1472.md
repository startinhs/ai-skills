# Plan Fix — Request 1472
**Sản phẩm ở eSales được chỉnh all field một cách độc lập (không bị SAP overwrite)**

## Thông tin
- Branch: `fixbug-issue-1472/tinhlm`
- Base: `release/1.0.0-avntt-rc1`
- Priority: Medium | Type: Request

## Yêu cầu
Bộ phận Master của AVN muốn có thể can thiệp (edit) thông tin sản phẩm trực tiếp ở eSales trong trường hợp urgent, đặc biệt là đơn vị quy đổi (UOM conversion), trước khi đơn hàng convert về SAP. Hiện tại, SAP sync sẽ overwrite tất cả fields.

## Phân tích hiện trạng

**File SAP Sync:**
`modules/hqsoft.sap.dmsintegration/src/HQSOFT.SAP.DMSIntegration.EntityFrameworkCore/Products/EfCoreProductInterfacesRepository.Extended.cs`

Method `UpdateProductAsync()` — overwrite toàn bộ:
- Code, Name, ShortName, Status, IsPA, HierarchyL01-L05, IsPromotion, CompanyCode
- CategoryL01-L05, SecondarySellingUOMCode, StockBasePrice, ListPrice
- Attribute01-03, ProductOrderName, TaxCode, BaseUOMCode, BOM, Weight
- **Không có bất kỳ flag IsManualOverride nào.**

## Design Options

### Option A — IsManualOverride flag (Recommended)
Thêm field `IsManualOverride` (boolean, default=false) vào `Products` table.

Khi `IsManualOverride = true`:
- SAP sync **bỏ qua** update cho product đó (hoặc chỉ bỏ qua một số fields được protect)
- User có thể toggle `IsManualOverride` từ màn hình Product detail trong eSales

**Pros:** Đơn giản, minimal impact, user control rõ ràng
**Cons:** Khi SAP có update quan trọng (VD: thay đổi giá, đổi status), eSales sẽ không nhận được

**Implementation effort:** Medium
- 1 migration: thêm column `IsManualOverride` vào `Products`
- 1 field trong Product entity
- 1 DTO update + UI toggle trong Product form
- SAP sync logic: skip UpdateProductAsync khi IsManualOverride=true

### Option B — Selective field protection
Thêm các flag riêng per-field group: `IsManualUOM`, `IsManualPrice`, `IsManualStatus`

**Pros:** Fine-grained control per field group
**Cons:** Phức tạp hơn, nhiều fields hơn

### Option C — OverrideUntilDate
Thêm `OverrideUntilDate` datetime. SAP sync bỏ qua khi `OverrideUntilDate > NOW()`.

**Pros:** Tự động hết hạn sau thời gian chỉ định
**Cons:** User phải nhớ set date, có thể bị bỏ qua

## Recommended: Option A (IsManualOverride)

## ⚠️ BLOCKING: Cần xác nhận trước khi implement

1. **Scope fields:** Cần protect toàn bộ fields hay chỉ UOM-related fields (`BaseUOMCode`, `SecondarySellingUOMCode`)?
2. **SAP override behavior:** Khi IsManualOverride=true, SAP sync bỏ qua hoàn toàn hay chỉ bỏ qua UOM fields?
3. **Expiry:** Override có hết hạn tự động không? Hay user phải toggle lại?
4. **Audit:** Có cần log khi SAP muốn update nhưng bị block bởi IsManualOverride?
5. **Scope của request:** Có cần làm ngay hay có thể schedule cho sprint sau?

## Implementation Steps (khi approved)
1. Migration: thêm `IsManualOverride boolean NOT NULL DEFAULT false` vào `Products`
2. Entity: thêm property vào `Product.cs`
3. DTO: thêm field vào Product DTOs
4. AppService: include trong create/update flow
5. UI: thêm toggle control trong Product form
6. SAP sync: wrap `UpdateProductAsync()` với check `!product.IsManualOverride`
7. Build + migrate + test

**Note:** Cần ask user build trước khi chạy migration.
