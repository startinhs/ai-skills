# Plan Fix — Issue 1575
**SKU: Lỗi khi kiểm tra trạng thái chỉnh sửa của grid SKU Detail**

## Thông tin
- Branch: `fix/fix-SKU-1575-tinhlm`
- Base: `release/1.0.0-avntt-rc1`
- Priority: Medium | Type: Issue

## Root Cause

Property `IsEditing` được truy cập trực tiếp trên `DxGrid` nhưng trong DevExpress Blazor 25.2, `IsEditing` đã thay đổi thành method `IsEditing()` (function call), không phải property. Truy cập `== true` vào property có thể trả về sai kết quả hoặc throw runtime exception khi grid chưa init.

**File:** `src/.../Pages/Inventory/SKU/SKU.razor.cs`

**Trước fix (SAI):**
```csharp
private bool HasSelectedSKUDetailWithZeroOrder =>
    SelectedSKUDetails.OfType<SKUDetailUpdateDto>().Any(IsSKUDetailOrderIncomplete)
    || (SKUDetailGrid?.IsEditing == true    // ← property access sai API
        && IsSKUDetailOrderIncomplete(EditingSKUDetail)
        && SelectedSKUDetails.OfType<SKUDetailUpdateDto>().Any(s => s.Idx == EditingSKUDetail!.Idx));
```

**Sau fix (ĐÚNG):**
```csharp
private bool HasSelectedSKUDetailWithZeroOrder =>
    SelectedSKUDetails.OfType<SKUDetailUpdateDto>().Any(IsSKUDetailOrderIncomplete)
    || (SKUDetailGrid != null && SKUDetailGrid.IsEditing()   // ← null-check + method call
        && IsSKUDetailOrderIncomplete(EditingSKUDetail)
        && SelectedSKUDetails.OfType<SKUDetailUpdateDto>().Any(s => s.Idx == EditingSKUDetail!.Idx));
```

## Fix

**File:** `src/HQSOFT.Xspire.Application.Blazor/Pages/Inventory/SKU/SKU.razor.cs`

Trong property `HasSelectedSKUDetailWithZeroOrder`:
1. Thay `SKUDetailGrid?.IsEditing == true` → `SKUDetailGrid != null && SKUDetailGrid.IsEditing()`
2. Thêm null-check tách biệt trước khi gọi method
