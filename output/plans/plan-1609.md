## Cơ chế hiện tại (Đã phân tích)

KPI visibility được kiểm soát bằng trường `Status` trong bảng `KPIDefinitions`:
- **Hiện thị:** `Status = 'Hoạt động'`
- **Ẩn:** `Status = 'Ngừng hoạt động'`

Hai PostgreSQL functions đều filter theo Status:
- `fs_rp_dailykpi.psql` — line 146: `WHERE kd."Status" = 'Hoạt động'`
- `fs_rp_dailykpi_3parms.psql` — line 146: `WHERE kd."Status" = 'Hoạt động'`

Snapshot job (mobile): cùng filter → ẩn KPI trên cả Web và App.

## Plan Fix

### Option A (Recommended): Update Status trong DB
Không cần code change. Chỉ cần biết danh sách KPI cần ẩn → update Status → tức thì có hiệu lực.

```sql
-- Ẩn các KPI cần ẩn (cần danh sách từ IT email):
UPDATE "KPIDefinitions"
SET "Status" = 'Ngừng hoạt động',
    "LastModificationTime" = NOW()
WHERE "Code" IN ('KPI_CODE_1', 'KPI_CODE_2', ...);

-- Kích hoạt KPI FY26 mới (nếu cần):
UPDATE "KPIDefinitions"
SET "Status" = 'Hoạt động',
    "LastModificationTime" = NOW()
WHERE "Code" IN ('KPI_FY26_CODE_1', ...);
```

### Option B: Thêm field ApplicableForPilot (nếu cần phân biệt Pilot vs Regular)
Nếu yêu cầu là: ẩn với non-pilot users, hiển thị với pilot users → cần thêm field mới.
→ Tốn thêm effort: migration, backend logic, snapshot job thay đổi.
