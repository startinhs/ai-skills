# Plan Fix — Issue 1600
**Hóa đơn EXCHANGE: tên khách hàng hiển thị sai và thiếu dòng ghi chú "Đổi hàng"**

## Thông tin
- Branch: `fix/fix-eInvoice-1600-tinhlm`
- Base: `release/1.0.0-avntt-rc1`
- Priority: High | Type: Issue

## Yêu cầu

Khách hàng nhóm EXCHANGE (GroupCode = "EXCHANGE") có tên dạng `"(Đổi hàng X-Y) TenKhachHang"` hoặc `"[BRACKET]prefix_Đổi hàng TenKhachHang"`. Khi in hóa đơn:
1. **CustomerName** phải hiển thị tên thực (bỏ prefix "Đổi hàng ...")
2. Phần prefix "Đổi hàng ..." phải được thêm vào cuối hóa đơn như một **dòng ghi chú** (IsSum=4, VatRate=-4)

## Root Cause

`SalesOrderToInvoiceMapper.cs` không có logic xử lý tên khách hàng EXCHANGE — dùng `CustomerName` nguyên si làm display name trên hóa đơn.

**File:** `modules/hqsoft.sap.dmsintegration/src/HQSOFT.SAP.DMSIntegration.Application/InvoiceMapping/Services/SalesOrderToInvoiceMapper.cs`

## Fix

**Bước 1 — Parse tên EXCHANGE:**
```csharp
string? exchangeNoteLabel = null;
var isExchangeCustomer = customer?.GroupCode?.Equals("EXCHANGE", StringComparison.OrdinalIgnoreCase) == true
    || customer?.CustomerGroupCode?.Equals("EXCHANGE", StringComparison.OrdinalIgnoreCase) == true;
if (isExchangeCustomer && TryParseExchangeCustomerName(customerDisplayName, out var parsedLabel, out var parsedActualName))
{
    exchangeNoteLabel = parsedLabel;
    customerDisplayName = parsedActualName;  // Chỉ dùng tên thực
}
```

**Bước 2 — Thêm dòng ghi chú cuối hóa đơn:**
```csharp
if (!string.IsNullOrWhiteSpace(exchangeNoteLabel))
{
    request.Items.Add(new InvoiceVatItemDto
    {
        ProductName = exchangeNoteLabel,
        Quantity = 0, UnitPrice = 0, Total = 0,
        VatRate = -4, IsSum = 4  // Dòng ghi chú đặc biệt
    });
}
```

**Bước 3 — Thêm `TryParseExchangeCustomerName` (static method):**

Hỗ trợ 3 format:
- `[BRACKET]prefix_Đổi hàng ...` → Regex: `^\[.*?\]\w*_`
- `(Đổi hàng X-Y) CustomerName` → Regex: `^\(đổi\s+hàng[^)]*\)\s*`
- `Đổi hàng [words] (X-Y) CustomerName` → Regex: `^(đổi\s+hàng[^(]*\([^)]*\))\s*`

## Phạm vi thay đổi
- **1 file:** `SalesOrderToInvoiceMapper.cs`
- **3 Regex patterns** (compiled, static) + 1 private method `TryParseExchangeCustomerName`
- Không cần migration
