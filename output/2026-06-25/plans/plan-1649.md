## Root Cause (Đã xác nhận qua DB + code)

**Đơn SO0000008177 có:**
- Sản phẩm: M-400G-L (140001714), TaxPercent=**10%**, CashBeforeTaxes=147,300,200, TaxAmount=14,730,020
- SalesOrderTradeDiscount: TradeDiscountCode=T_TT20260501, PaymentAmount=**100,000**, TaxAmount=**8,000** (rate 8%)

**Bug trong mapper — tính toán tax bracket:**

```csharp
// vat8Items = products WHERE TaxPercent=8 → EMPTY (chỉ có 10% products)
var vat8Items = products.Where(p => p.TaxPercent == 8); // Empty!
tradeDiscountByVat.TryGetValue(8, out var tradeDiscountGross8); // = 100,000

var grossValue8 = vat8Items.CashBeforeTaxes - tradeDiscountGross8;
// = 0 - 100,000 = -100,000 ← ÂM!

var vatAmount8 = vat8Items.TaxAmount - tradeDiscountGross8 * 8% / 100;
// = 0 - 8,000 = -8,000 ← ÂM!

// Điều kiện hiện tại: if (grossValue8 != 0 || vatAmount8 != 0) → TRUE → gửi giá trị ÂM lên VNPT!
```

Nguyên nhân gốc: CKTM có tax rate 8% (từ BonusLine.TaxPercent) nhưng đơn chỉ có sản phẩm 10% VAT → bracket 8% trống nhưng vẫn bị subtract → âm.

## Fix

**File:** `SalesOrderToInvoiceMapper.cs` — phần tính bracket VAT cho mỗi tax rate (lines ~1154–1226)

Thay đổi condition cho **tất cả 4 brackets** (0, 5, 8, 10):

```csharp
// TRƯỚC (cho mỗi bracket, ví dụ vat8):
if (grossValue8 != 0 || vatAmount8 != 0)
{
    request.GrossValue8 = grossValue8;
    request.VatAmount8 = vatAmount8;
}

// SAU — chỉ output bracket khi có sản phẩm hoặc grossValue dương:
var hasVat8Products = vat8Items.Any(p => (decimal)(p.CashBeforeTaxes) > 0);
if (hasVat8Products && (grossValue8 != 0 || vatAmount8 != 0))
{
    request.GrossValue8 = grossValue8;
    request.VatAmount8 = vatAmount8;
}
```

Áp dụng tương tự cho: `vat0`, `vat5`, `vat10` (pattern nhất quán).

## Giải thích logic
- Nếu không có sản phẩm ở tax rate X: bracket X không được xuất ra VNPT
- CKTM tại tax rate 8% trên đơn có 100% sản phẩm 10% → bracket 8% bị bỏ qua
- Discount line item (IsSum=2) vẫn hiển thị trên hóa đơn, nhưng không tạo bracket âm
- **Trade-off:** TaxAmount của CKTM (8,000đ) không được khấu trừ khỏi header tax → tổng thuế trên invoice cao hơn 8,000đ. Đây là acceptable khi data CKTM có tax rate sai lệch so với sản phẩm.
